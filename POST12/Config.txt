LAB 12 — ENTERPRISE CAMPUS LAYER 2 CAPSTONE

This version uses a proper three-tier campus design and avoids topology mistakes. Every redundant link has a purpose, STP has a clear role, and EtherChannel is used where enterprises actually deploy it.

Build a campus Layer 2 network using:

SW1 CONFIGURATION (CORE PRIMARY)
enable
conf t

hostname SW1

vtp domain CAMPUS
vtp mode server

vlan 10
 name USERS

vlan 20
 name SERVERS

vlan 99
 name MANAGEMENT

spanning-tree vlan 1,10,20,99 root primary

interface range g0/1-2
 switchport mode trunk
 channel-group 1 mode active
 no shutdown

interface port-channel1
 switchport mode trunk

interface g0/3
 switchport mode trunk
 no shutdown

end

FAILURE TESTS
Test 1 — EtherChannel Member Failure
SW1(config)#interface g0/1
SW1(config-if)#shutdown

Expected:

Po1 remains UP
Traffic unaffected
Test 2 — Access Uplink Failure
SW5(config)#interface g0/1
SW5(config-if)#shutdown

Expected:

STP activates backup uplink
Traffic continues
Test 3 — Core Failure

Power off SW1.

Expected:

SW2 becomes Root Bridge

Campus remains operational
