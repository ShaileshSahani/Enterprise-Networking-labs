🔧 Lab 01/50 — STP Root Bridge Manipulation & Path Analysis

Built a 6-switch redundant Layer 2 topology using Cisco 2960 switches to analyze real STP behavior under topology changes.

━━━━━━━━━━━━━━━━━━━━━━

Topology:
• 6 switches in a ring
• Multiple redundant paths
• Dual uplinks between distribution/core
• VLAN 1 and VLAN 10 trunked throughout the network

A topology where improper Layer 2 design can easily create switching loops.

━━━━━━━━━━━━━━━━━━━━━━

Phase 1 — Default STP Election

No bridge priorities configured initially.

Observed:
• Lowest MAC address elected as Root Bridge
• STP automatically blocked redundant links
• Entire forwarding topology recalculated around the elected root

Verified:
• Root Ports
• Designated Ports
• Blocking Ports
• Active forwarding paths

━━━━━━━━━━━━━━━━━━━━━━

Phase 2 — Root Bridge Manipulation

Configured:
• S1 as Root Primary
• S2 as Root Secondary

After convergence:
• Root ports changed
• Blocking ports shifted
• Forwarding paths recalculated
• Traffic flow through the topology changed completely

This demonstrates how STP directly influences Layer 2 traffic engineering — not just loop prevention.

━━━━━━━━━━━━━━━━━━━━━━

Verification Performed:
• show spanning-tree analysis
• Root bridge validation
• Blocked-port identification
• MAC address path tracing
• Continuous ping testing
• STP convergence observation

Result:
• Stable forwarding topology
• No packet loss after convergence
• Successful failover recalculation

━━━━━━━━━━━━━━━━━━━━━━
