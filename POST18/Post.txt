🔧 Lab 18/50 — HSRP with ISP Failover & Interface Tracking

Built an enterprise high-availability lab today combining HSRP gateway redundancy with WAN failure detection.

What was configured?

✅ HSRP Active/Standby

✅ Virtual Gateway

✅ Interface Tracking

✅ ISP Failover

✅ Automatic Recovery

Topology

• R1 connected to ISP1

• R2 connected to ISP2

• Both routers provided gateway redundancy for VLAN 10

• HSRP VIP: 10.10.10.254

Initial State

• R1 Priority = 120

• R2 Priority = 100

Result:

✅ R1 became Active

✅ R2 became Standby

All clients used the virtual gateway instead of physical router IPs.

Interface Tracking

Configured on R1:

standby 10 track g0/1 30

If the ISP-facing interface fails, R1 automatically loses 30 priority points.

Failover Test

Shutdown R1 WAN interface.

Result:

• R1 Priority: 120 → 90

• R2 Priority: 100

✅ R2 automatically became Active

✅ Gateway remained available

✅ Client connectivity continued

Recovery Test

Restored the WAN link on R1.

Because preemption was enabled:

standby 10 preempt

✅ R1 automatically reclaimed the Active role

✅ R2 returned to Standby

Key Learning

HSRP alone protects against router failure.

HSRP + Interface Tracking protects against WAN failure.

Even if the router stays online, loss of Internet connectivity can trigger failover to a healthier gateway.

This is a common design in enterprise branches, datacenters, and campus networks where uptime is critical.
