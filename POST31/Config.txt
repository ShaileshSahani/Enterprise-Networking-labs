🔧 LAB 31/50 — GRE Tunnel Between Branches

Since your LAB 30 was an Enterprise WAN through an ISP Cloud, the next logical step is to build a GRE overlay across that provider network.

Objective

Build a GRE Tunnel between HQ and Branch.

Underlay = ISP Network
Overlay = GRE Tunnel
Run OSPF over GRE
Exchange LAN Routes
Verify end-to-end reachability
📌 Topology

ISP Router

Simple routing only.

ISP Configuration
enable
conf t

hostname ISP

interface g0/0
 ip address 10.1.1.2 255.255.255.252
 no shutdown

interface g0/1
 ip address 20.1.1.2 255.255.255.252
 no shutdown

ip route 192.168.1.0 255.255.255.0 10.1.1.1
ip route 192.168.2.0 255.255.255.0 20.1.1.1

end
wr
HQ Router (R1)
enable
conf t

hostname R1

interface g0/0
 ip address 192.168.1.1 255.255.255.0
 no shutdown

interface g0/1
 ip address 10.1.1.1 255.255.255.252
 no shutdown

ip route 0.0.0.0 0.0.0.0 10.1.1.2
GRE Tunnel
interface Tunnel0

 ip address 172.16.12.1 255.255.255.252

 tunnel source g0/1

 tunnel destination 20.1.1.1

 no shutdown
OSPF over GRE
router ospf 1

 router-id 1.1.1.1

 network 192.168.1.0 0.0.0.255 area 0

 network 172.16.12.0 0.0.0.3 area 0
end
wr
Branch Router (R2)
enable
conf t

hostname R2

interface g0/0
 ip address 192.168.2.1 255.255.255.0
 no shutdown

interface g0/1
 ip address 20.1.1.1 255.255.255.252
 no shutdown

ip route 0.0.0.0 0.0.0.0 20.1.1.2
GRE Tunnel
interface Tunnel0

 ip address 172.16.12.2 255.255.255.252

 tunnel source g0/1

 tunnel destination 10.1.1.1

 no shutdown
OSPF
router ospf 1

 router-id 2.2.2.2

 network 192.168.2.0 0.0.0.255 area 0

 network 172.16.12.0 0.0.0.3 area 0

end
