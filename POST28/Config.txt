🔷 ASA Base Configuration
enable

conf t

interface ethernet0/0
 nameif outside
 security-level 0
 ip address 200.1.1.1 255.255.255.0
 no shut

interface ethernet0/1
 nameif inside
 security-level 100
 ip address 192.168.1.1 255.255.255.0
 no shut
Create ACL

Allow Ping:

access-list OUTSIDE-IN extended permit icmp any any

Allow HTTP:

access-list OUTSIDE-IN extended permit tcp any any eq 80

Block Telnet:

access-list OUTSIDE-IN extended deny tcp any any eq 23

Apply ACL:

access-group OUTSIDE-IN in interface outside

Save:

write memory
Verification

Check ACL:

show access-list

Check Access Group:

show access-group

Check Interface:

show running-config

Firewall Concepts
ACL

A list of permit and deny statements.

Access Group

Binds ACL to an interface.

access-group OUTSIDE-IN in interface outside

Without this command:

ACL exists

BUT

does nothing
Stateful Firewall

ASA remembers connections.

Inside → Outside

Return traffic automatically allowed

This is one reason firewalls are smarter than routers with ACLs.
