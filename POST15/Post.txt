Enterprise Scenario

Not every device requires the same type of Internet access.

Some systems need a permanent public IP.

Examples:

• Web Servers
• Application Servers
• CCTV Systems
• Remote Management Devices

Other users simply need temporary outbound connectivity.

This is where Static NAT and Dynamic NAT are used together.

━━━━━━━━━━━━━━━━━━━━━━

Lab Design

• R1 = NAT Router
• R2 = Server-Side Router
• LAN1 = Static NAT Users
• LAN2 = Dynamic NAT Users
• Server Network behind R2

━━━━━━━━━━━━━━━━━━━━━━

Static NAT Configuration

Configured 1:1 address mapping.

Each internal device received its own dedicated public IP.

Result:

• Same public IP every time
• Predictable connectivity
• Easy external access

Perfect for business-critical devices.

━━━━━━━━━━━━━━━━━━━━━━

Dynamic NAT Configuration

Created a NAT pool containing multiple public addresses.

Internal users from LAN2 dynamically borrowed available addresses from the pool.

Result:

• Public IP assigned only when needed
• Efficient address utilization
• Automatic allocation and release

━━━━━━━━━━━━━━━━━━━━━━

Routing Configuration

Configured static routes between routers.

Verified:

• Internal LAN reachability
• Transit network communication
• Access to server subnet

Key Learning

Static NAT:

• One-to-one mapping

• Permanent public address

• Best for servers and critical systems

Dynamic NAT:

• Many internal hosts

• Pool-based translation

• More efficient public IP usage
